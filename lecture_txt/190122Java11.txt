자바 11일차 강의 정리
-------------------------------------------------------------------------------------------------------------------------------------

자바는 단일상속이 원칙, 다중상속이 불가함
이를 대체하기 위해 인터페이스를 사용해서 다중 상속 처럼 사용

인터페이스 (Interface)
구체적인 바디를 구현해 놓은 것이 아닌 틀만 제공
implement
인터페이스 내부 메소드는 항상 public

subclassing - 부모자식간의 상속관계
subtyping - 자식과 인터페이스간의 연결관계

instanceof - 타입 비교

인터페이스에 선언한 메소드와 상속받은 부모의 메소드가 동일하면 오버라이딩으로 확인하고 사용가능
-------------------------------------------------------------------------------------------------------------------------------------

Inner Class
메소드 혹은 클래스 내부에서 만들어지는 내부 클래스

익명 클래스 (Anonymous Class)
타입만 가지고 쓰는 클래스
새로운 타입으로 일회용 객체를 정의 하며 만드는 클래스
일회성으로 사용
모든 클래스는 익명 클래스로 사용 가능
-------------------------------------------------------------------------------------------------------------------------------------

람다
익명함수
인터페이스 내부에 구현해야될 메소드가 하나밖에 없을 경우

메소드가 하나인 인터페이스 -> 함수형 인터페이스

interface Nice {
  public void sayNice();
}
  
Nice n1 = ()->System.out.println("헐~~~~~~");
n1.sayNice();
